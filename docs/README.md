# Documentation

Reference materials and project documentation.

## Directory Structure

| Directory | Purpose |
|-----------|---------|
| `emails/` | Email correspondence related to the project (coauthors, editors, TAs) |
| `papers/` | Reference papers, working papers, and PDFs used as input for slides or analysis |
| `technical/` | Technical documentation, software guides, API references, and tool setup notes |

## Guidelines

- `papers/` is for reading/reference, not for your own paper drafts (those go in `output/papers/`)
- Keep reference PDFs here — the `pdf-processing` rule governs how Claude reads them
- Email threads can be useful for capturing design decisions and feedback
